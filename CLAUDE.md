# General Workflow

Before implementing any feature:

1. Review the existing architecture.
2. Read the current implementation before making changes.
3. Produce a concise implementation plan.
4. Modify only files related to the requested feature.
5. Never rewrite unrelated code.
6. Preserve backward compatibility.
7. Explain any required database schema changes before implementing them.
8. Follow the Repository Pattern and Service Layer.
9. Update documentation after completing the feature.
10. Stop after completing the requested module and wait for approval.
